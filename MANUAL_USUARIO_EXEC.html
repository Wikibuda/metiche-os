Manual Ejecutivo - metiche-os

Version corta para operacion y onboarding rapido.

Fase 4MulticanalNarrativaPlane

Que hace el sistema

  1. Ejecuta tareas y decide ruta.
  2. Valida por canal (Telegram, WhatsApp, Shopify, Dashboard, Deepseek).
  3. Registra eventos narrativos y genera cronicas.
  4. Sincroniza fallas a Plane cuando esta habilitado.

Comandos minimos (dia a dia)

  • metiche run --task "..." --task-type whatsapp --description "..."
  • metiche validate --task-id <task_id>
  • metiche narrator-tick --limite 100
  • metiche --cuentame --limite 5
  • metiche build-bitacora

Variables criticas

  • DATABASE_URL
  • OPENCLAW_GATEWAY_URL (default http://127.0.0.1:18797)
  • PLANE_SYNC_ENABLED
  • Credenciales de canal (Telegram, Shopify, etc)

Prioridad de .env: ~/.openclaw/workspace/.env -> .env -> ~/.openclaw/.env.

WhatsApp (check completo)

  1. Gateway: GET {OPENCLAW_GATEWAY_URL}/health debe devolver ok=true o status=live.
  2. Canal: openclaw channels list | grep -q "WhatsApp default: linked, enabled" debe retornar 0.

Ambos checks son obligatorios para estado passed.

Escala de asombro

  • 5: falla critica
  • 4: validacion exitosa
  • 3: informativo/no critico

Cuando algo falla

  • Revisar variables en ~/.openclaw/workspace/.env.
  • Probar gateway: curl http://127.0.0.1:18797/health.
  • Verificar canal: openclaw channels list.
  • Re-ejecutar: validate y luego narrator-tick.

Extender con nuevo validador

  1. Crear clase en app/domain/validators/ heredando BaseValidator.
  2. Registrar en __init__.py y en _validator_registry().
  3. Mapear task_type en _resolve_required_channels(...).
  4. Probar flujo completo y documentar variables.
