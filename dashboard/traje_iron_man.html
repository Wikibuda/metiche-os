¿Qué es el Traje Iron Man?

Una metodología probada para operaciones masivas en Plane

El Traje Iron Man es un sistema de procesamiento masivo diseñado para ejecutar operaciones a gran escala en Plane (sistema de gestión de proyectos) con precisión quirúrgica y trazabilidad total.

Precisión Quirúrgica

Procesamiento por lotes controlados con validación entre cada lote y capacidad de rollback.

Bitácora Narrativa

Reutilizable

    Consulta Solo Lectura

    Análisis estratégico sin modificar la base de datos

    Identificar candidatos usando SQL de solo lectura con CTE + JOINS. Generar reporte analítico con métricas clave.

    SQL de Análisis (Ejemplo) 
    -- Identificar issues LOW obsoletos (>30 días sin actividad)
WITH candidatos AS (
    SELECT 
        i.id,
        i.sequence_id,
        i.name,
        s.name as estado,
        EXTRACT(DAY FROM NOW() - i.updated_at) as dias_inactivos
    FROM plane_issue i
    JOIN plane_state s ON i.state_id = s.id
    JOIN plane_issue_label il ON i.id = il.issue_id
    JOIN plane_label l ON il.label_id = l.id
    WHERE 
        s.name IN ('backlog', 'unstarted')
        AND l.name = 'low'
        AND i.archived_at IS NULL
        AND EXTRACT(DAY FROM NOW() - i.updated_at) > 30
    ORDER BY i.updated_at ASC
    LIMIT 100
)
SELECT * FROM candidatos;

    UPDATE por Lotes

    Precisión quirúrgica con rollback capacity

    Procesar candidatos en lotes controlados (default: 20 issues por lote) con validación entre lotes.

    SQL de UPDATE por Lotes 
    WITH batch AS (
    SELECT id
    FROM plane_issue
    WHERE id IN (
        -- Subquery idéntica a Fase 1
        SELECT i.id
        FROM plane_issue i
        JOIN plane_state s ON i.state_id = s.id
        WHERE s.name IN ('backlog', 'unstarted')
        AND l.name = 'low'
        LIMIT 20  -- ¡LOTE PEQUEÑO!
    )
    FOR UPDATE SKIP LOCKED
)
UPDATE plane_issue i
SET 
    state_id = (SELECT id FROM plane_state WHERE name = 'completed'),
    updated_at = NOW(),
    completed_at = NOW()
FROM batch b
WHERE i.id = b.id
RETURNING i.id, i.sequence_id, i.name;

    ⚙️ Opciones del CLI

    --mision (requerido)

    Nombre descriptivo de la misión. Aparece en bitácoras y logs.

    --lote (default: 20)

    Tamaño de lote para procesamiento. Recomendado: 20 issues por lote.

    --dry-run

    Solo análisis, sin cambios reales. Perfecto para validación previa.

    --plantilla

    Plantilla predefinida: limpieza o archivado.

    --analisis / --update

    SQL personalizado para análisis y actualización.

    --validacion

    SQL personalizado para validación post-operación.
