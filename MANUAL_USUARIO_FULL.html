1) Objetivo del sistema

  • Ejecucion de tareas con ruteo y validacion por canal.
  • Sincronizacion de fallas de validacion hacia Plane (si esta habilitado).
  • Narrativa operacional: cronicas, resumenes, momentos y candidatos.
  • Proyeccion de bitacora en Markdown para lectura humana.
  1. Se crea/ejecuta una tarea.
  2. Se decide ruta y se registra ejecucion.
  3. Se valida por canal y se emiten validation_attempt.
  4. El selector narrativo promueve eventos a cronicas.
  5. Se exporta bitacora a archivo Markdown.

2) Arquitectura por modulos

  • app/main.py: inicializa FastAPI y routers.
  • app/api/: endpoints HTTP.
  • app/cli/: comandos de operacion.
  • app/core/: config y base de datos.
  • app/domain/tasks/: flujo de tareas.
  • app/domain/validators/: validadores por canal.
  • app/domain/narrative/: selector y servicio narrativo.
  • app/integrations/plane.py: cliente Plane.
  • app/projections/bitacora.py: export markdown.

Flujo de datos: Task -> Decision -> Execution -> Validation, eventos en task_events, promocion narrative_candidates -> narrativeentry.

4) Comandos CLI

Se pueden ejecutar como metiche ... o python -m app.cli.main ....

  • metiche init-db
  • metiche run --task "titulo" --task-type whatsapp --description "detalle"
  • metiche validate --task-id <task_id>

6) Validadores por canal

Implementados: Telegram, WhatsApp, Dashboard, Shopify y Deepseek.

  • Los canales requeridos se resuelven por task_type, marcador en descripcion [channels=...] y VALIDATION_REQUIRED_CHANNELS.
  • Cada validador retorna ValidationResult.
  • Cada resultado emite task_event tipo validation_attempt.

WhatsApp (Fase 4)

  1. Health gateway: GET {OPENCLAW_GATEWAY_URL}/health con timeout 5s.
  2. Canal habilitado: openclaw channels list | grep -q "WhatsApp default: linked, enabled".

Solo pasa si ambos checks son exitosos. Tiene cache TTL de 60s.

7) Como agregar un nuevo validador

  1. Crear app/domain/validators/mi_canal_validator.py.
  2. Heredar de BaseValidator e implementar validate(...).
  3. Retornar ValidationResult consistente.
  4. Exportar en app/domain/validators/__init__.py.
  5. Registrar en _validator_registry().
  6. Mapear task_type -> canal en _resolve_required_channels(...).
  7. Probar con run, validate, narrator-tick y --cuentame.

8) Bitacora y narrativa

  1. Ejecutar tareas.
  2. Correr selector: metiche narrator-tick --limite 100.
  3. Revisar: metiche --cuentame --limite 10.
  4. Exportar: metiche build-bitacora.

Ruta de salida: projections/bitacora/bitacora_de_asombros.md.

Escala de asombro en validaciones: 5 (falla critica), 4 (exito), 3 (informativo/no critico).

9) Plane

Si PLANE_SYNC_ENABLED=true y hay fallas de validacion:

  • Se crea issue.
  • Se comenta resumen de canales fallidos.

10) Troubleshooting

  • Credenciales faltantes: revisar ~/.openclaw/workspace/.env.
  • WhatsApp no conecta: validar OPENCLAW_GATEWAY_URL y probar curl http://127.0.0.1:18797/health.
  • No hay cronicas: correr metiche narrator-tick --limite 100.
  • DB inaccesible: revisar DATABASE_URL absoluta y permisos.
